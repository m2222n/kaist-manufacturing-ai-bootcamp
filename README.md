## 프로젝트 목표

산업 현장에서 출력한 3D 프린터 부품(27종)을, **다양한 환경에서도 실제 카메라가 정확하게 인식**하도록 만드는 비전 AI 모델을 개발한다. 궁극적으로는 빈(bin) 안의 부품을 인식해 로봇이 집을 수 있는 **3D 좌표(6DoF pose)** 까지 추정하는 것이 목표다.

- 단순 평가셋 지표(mAP 등)는 중간 신호일 뿐, **실제 카메라로 부품을 정확히 인식**하는 것이 종착점이다.
- 데이터셋 구축 → 전처리 → 모델링 → 검증까지 한 사이클을 6주 안에 완주한다.
- 산출물(데이터·모델·코드)은 발표 이후 실제 산업 적용까지 이어가는 것을 염두에 둔다.

### 현재 접근 방식 (1차 발표 피드백 반영, 6/9~)

- **사전 정의된 27부품 지도학습 우선** (제로샷/retrieval보다 supervised 먼저)
- **합성데이터(디지털 트윈)로 학습셋 자동 생성** — 실물 촬영·수동 라벨링 인력 한계를 합성으로 보완
- **모달리티 분담**: Depth(ToF) / Segmentation(RGB). depth 정보만으로도 성능이 더 나올 수 있어 별도 실험
- **실사 test set** 으로 정직하게 평가 (학습데이터도 점차 실사에 근접)

## 합성데이터 파이프라인 (`synth/`)

CAD(STL) 27부품을 가상 빈에 **중력으로 낙하·적재**시켜 장면을 만들고, **RGB / Depth / Segmentation / 6DoF pose 라벨을 자동 생성**한다. 카메라·실물 없이 코드만으로 라벨링된 학습 데이터를 만드는 것이 핵심이다.

> 원본 CAD는 28종이었으나, 1종(`10_guide_paper_roll_l`, 약 236mm)은 보유 3D 프린터(Form 4) 빌드볼륨을 초과해 실물 출력이 불가능하여 데이터셋 대상에서 제외했다. 따라서 학습·합성데이터 대상은 **27종**이다.

- **도구**: [BlenderProc](https://github.com/DLR-RM/BlenderProc) (Blender 4.2 물리엔진 + 멀티모달 렌더)
- **장면당 자동 출력**: `colors`(RGB) / `depth`(meter) / `instance_segmaps` / `category_id_segmaps` / 6DoF pose — **수동 라벨링 0**

### 데이터셋 버전

| 버전 | 부품 | 배경 | 의도 |
|------|------|------|------|
| v1 | 회색 단색 | 단색 | 기본 형상 |
| v2 | 색·텍스처 랜덤 | 도메인 랜덤화(텍스처/조명) | 실제 배경 미확정 대응 |
| v2_noisy | v2 동일 | v2 + ToF depth 노이즈 | depth 현실화 (실측 ToF 닮게) |
| **v3** | **회색 단색 고정** | **2종: 투명 플라스틱 박스(적재형) / 흰 책상(정렬형)** | **부품을 색이 아닌 형태로 판별 + 실제 빈피킹 2시나리오 정합** |

> v3는 부품을 회색 단색으로 고정한다 — 부품 색이 다양하면 모델이 형태가 아니라 색으로 판별해버릴 위험이 있기 때문이다. 배경만 2종(적재형/정렬형)으로 다양화한다.

### `synth/` 파일 안내

| 파일 | 역할 |
|------|------|
| `gen_one.py` / `run_batch.sh` | v1 — 단색 배경, 1 scene 단위 생성(상태 누수 회피) + 배치 |
| `gen_one_v2.py` / `run_batch_v2.sh` | v2 — 도메인 랜덤화(CC0 텍스처·재질·조명) |
| `gen_one_v3.py` / `run_batch_v3.sh` | **v3 — 회색 부품 + 배경 2종(투명박스 적재형 / 흰책상 정렬형)** |
| `depth_noise.py` | Blaze ToF depth 노이즈 모델 (거리비례 Gaussian + dropout + flying pixel) |
| `extract_all.py` / `extract_preview.py` | hdf5 → 4패널 PNG(RGB·depth·instance·category) 추출, 육안 검증·시연용 |
| `run_png_export.sh` | 데이터셋 일괄 PNG 추출 (공유용) |
