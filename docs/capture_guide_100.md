# Blaze 실증 본촬영 100장 — 촬영 가이드 (3그룹 × 9종)

> 부품이 **종류당 1개씩**(총 27개) + 큰 부품은 한 판에 다 안 들어감
> → 27종을 **3등분(9종씩)**, 그룹당 ~33장. 부품 교체 2번. 27종 각 33~34회 등장(완전 균등).
> label 번호 = `label_map_27parts.md` (반드시 이 표와 동일).

## 그룹 구성

| 그룹 | 부품 (label) | 부품 이름 | 장수 |
|------|-------------|-----------|------|
| **g1** | 1 ~ 9 | sol_block_a/b/front/back, guide_paper_l, r_guide_a, guide_paper_r, sw_block, variant | **34장** |
| **g2** | 10 ~ 18 | x2_bcf8ccb4, 14_13, roller_bracket, cam_f_bracket, mks_holder, button_function_niro, bracket_case, bracket_sen_1, bracket_sensor1 | **33장** |
| **g3** | 19 ~ 27 | bracket_sensor2, brkt_switch, roll_cover_left/right, main_body, plate_e, r_guide_a_l/r, top_inner_sheet | **33장** |

→ 합계 100장. 한 그룹 = 9종을 한 판에 다 깔고(각 1개씩) 촬영.

## 한 그룹(약 33장) 안에서 — 겹침 60% + 펼침 40%
- **겹침(적재형) ~20장**: 9개를 서로 겹쳐 쌓되 **너무 깊게 X**(맨 위만 잡힘). 합성 주력·실제 빈피킹 모습.
- **펼침(정렬형) ~13장**: 9개를 안 겹치게 드문드문. 형태 전체가 보이는 케이스.

## 공통 규칙 (매 장)
- 카메라 = 부품 위 **top-down**, 화면에 **부품만 + 배경 검정(NaN)**. 박스벽 들어오면 거리/각도 조정.
- 매 장 **위치·자세·뭉침·회전 다르게**. 같은 장면 두 번 X. `s` 누르기 전 손으로 흐트러뜨리기.
- `[OK]` 뜨면 좋음. `[..]`여도 **부품 형태 또렷 + 배경 검정**이면 저장 OK.
- 9개라 한 판이 좀 넓을 수 있음 → 9개가 화면에 다 들어오게 카메라 높이/위치 맞춤.
- 저장 = `s`(자동 번호). **그룹 바꿀 때 `스페이스`**(g1→g2→g3, 파일명에 기록).
- 저장 포맷 = raw(848×480 uint16 mm). 변환은 나중(조교 컨펌 후).

## 진행 순서 (Mac)
1. 실행: `cd ~/Desktop && sudo python blaze_capture_100.py --ip <BLAZE_IP> --scale 2`
2. **g1 부품(1~9) 9개 다 깔기** → 겹쳐서 ~20장(매번 흐트러뜨림) → 펼쳐서 ~13장 = 34장
3. **`스페이스`** 눌러 g2로 → **부품 교체(10~18)** → 같은 방식 33장
4. **`스페이스`** 눌러 g3로 → **부품 교체(19~27)** → 33장
5. 화면 `saved=NN/100` 보며 진행. 100장 차면 알림.
