# Blaze 실증 본촬영 100장 — 촬영 가이드 (구성 그룹 10개)

> 부품 100번 교체는 비현실적 → **구성 그룹 10개**로 묶어, 한 그룹 깔고 10장씩 연사.
> 부품 교체는 10번만. 27종은 각 3~4회 등장(공정). 겹침 60% + 펼침 40%.
> label 번호 = `label_map_27parts.md` (반드시 이 표와 동일).

## 공통 규칙 (매 장)
- 카메라 = 부품 위 **top-down**, 화면에 **부품만 + 배경 검정(NaN)**. 박스벽 들어오면 거리/각도 조정.
- 매 장 **위치·자세·뭉침·회전 다르게**. 같은 장 두 번 찍지 말 것.
- `[OK]` 뜨면 좋음. `[..]`여도 **부품 형태 또렷 + 배경 검정**이면 저장 OK.
- 저장 = `s` (자동 번호). **구성 바꿀 때 `스페이스`**(그룹 번호 올라감 → 파일명에 기록).
- 저장 포맷 = raw(848×480 uint16 mm). 변환은 나중(조교 컨펌 후).

## 겹침 vs 펼침 (그룹마다 둘 다)
- **겹침(적재형) 6장**: 부품들 서로 겹쳐 쌓되 **너무 깊게 X**(맨 위만 잡힘). 합성 주력·실제 빈피킹 모습.
- **펼침(정렬형) 4장**: 부품들 안 겹치게 드문드문. 형태 전체가 보이는 케이스.

## 진행 순서 (Mac)
1. 실행: `cd ~/Desktop && sudo python blaze_capture_100.py --ip <BLAZE_IP> --scale 2`
2. **g1 부품(1,2,3) 깔기** → 겹쳐서 6장(`s`×6, 매번 흐트러뜨림) → 펼쳐서 4장(`s`×4)
3. **`스페이스`** 눌러 g2로 → **부품 교체(4,5,6)** → 같은 방식 10장
4. g3~g10 반복. 화면 `saved=NN/100` 보며 진행.
5. 100장 차면 알림. 부족분·약한 부품 더 찍어도 됨(이어찍기 자동).
