## 문제 정의

박스(빈) 안에 여러 종류(27종)의 부품이 겹쳐 쌓여 있을 때, **각 부품이 어디에 있고 어느 부품인지**를 인식해 로봇이 집을 수 있게 한다. 어려움은 부품이 많고, 형상이 유사하며, 서로 겹치고(occlusion), 자세가 다양하다는 점이다.

### 핵심 강점 — 데이터를 모으지 않아도 되는 인식 모델

1. **데이터 수집·라벨링 불필요** — CAD만 있으면 합성으로 학습 데이터와 정답(GT)이 자동 생성된다. 실촬영 데이터 수집·라벨링은 그 자체로 가장 많은 시간이 드는 과정인데, 이를 통째로 건너뛰고 신규 부품 추가도 쉽다.
2. **Depth-only** — 색·조명 등 외부 환경에 강건하다. 회색 단색 부품도 형상으로 구분되어 합성↔실제 도메인 갭이 작다.
3. **CAD 연계** — 검출에서 끝나지 않고 '어느 CAD 부품인지'까지 식별해, 로봇이 부품별로 다르게 집는 단계로 바로 이어진다.

> 1차 발표(6/9) 피드백으로 방향이 수렴했다: RGB→**Depth만**, 분류→**검출+분할+CAD 식별**, 실촬영→**합성 데이터(GT 자동)**, 제로샷→**사전 정의 부품 지도학습 먼저**.

## 모델 구조 — 두 인코더 + 하나의 검출기

```
① 오프라인 — CAD 형상 사전 구축
   CAD STL(27종) → Point Cloud(8,192점, xyz+법선) → 3D CAD Encoder(PointNet++)
                 → CAD Memory Bank (27종 256-d 형상 임베딩 codebook, frozen)

② 추론 — Depth 한 장
   합성 Depth 장면(배경 NaN, 512²) → 2D Depth Detector(DETR/Mask2Former 계열)
                 → box(위치) / mask(분할) / class(1..27) / CAD VQ(어느 CAD)
                   └ CAD VQ = 쿼리 임베딩을 frozen codebook과 cosine 유사도로 매칭
```

- **3D CAD Encoder ([PointNet++](https://arxiv.org/abs/1706.02413))** — CAD를 점으로 뿌려 계층적 set abstraction으로 국소→전역 형상을 학습, 부품마다 형상 임베딩을 만들어 codebook에 저장. contrastive 학습으로 같은 CAD는 가깝게·다른 CAD는 멀게.
- **2D Depth Detector (DETR/Mask2Former 계열)** — Depth 한 장 입력(정규화 depth + valid mask, RGB·마스크 입력 없음). CNN+FPN → Transformer object query decoder(쿼리 100) → 4개 헤드(class/box/mask/CAD VQ). 추론 시 입력은 오직 Depth이며, 이는 실제 카메라 한 대로 배포하는 상황과 동일하다.
- **2단계 학습** — ① 검출(class+box+mask) 안정화(warmup) → ② CAD codebook 정렬(VQ joint) 추가. 손실 = class + box(L1+GIoU) + mask(BCE+Dice) + CAD(CE+align), Hungarian 매칭.

> 역할 분담: **모델 설계·고도화 = 전담 조교 / 데이터 구축·실증 구동·비교 실험 = 팀 6**.

## 합성데이터 파이프라인 (`synth/`)

CAD(STL) 27부품을 가상 빈에 **중력으로 낙하·적재**시켜 장면을 만들고, **Depth / Segmentation / 6DoF pose 라벨을 자동 생성**한다. 카메라·실물 없이 코드만으로 라벨링된 학습 데이터를 만드는 것이 핵심이다.

> 원본 CAD는 28종이었으나, 1종(`10_guide_paper_roll_l`, 약 236mm)은 보유 3D 프린터 빌드볼륨을 초과해 실물 출력이 불가능하여 데이터셋 대상에서 제외했다. 따라서 학습·합성데이터 대상은 **27종**이다.

- **도구**: [BlenderProc](https://github.com/DLR-RM/BlenderProc) (Blender 4.2 물리엔진 + 멀티모달 렌더)
- **장면당 자동 출력**: `depth`(meter, 배경 NaN) / `instance_segmaps` / `category_id_segmaps`(1..27) / 6DoF pose — **수동 라벨링 0**
- 부품은 **회색 단색**으로 고정한다 — 부품 색이 다양하면 모델이 형태가 아니라 색으로 판별해버릴 위험이 있기 때문이다.

### 학습 데이터셋 & split

- 학습용 **1,000 scene** (depth 512×512, crop 8,181개 = 부품 1개당 1 crop)
- **scene 단위 800 / 100 / 100 (train/val/test)** — crop(부품) 단위로 쪼개면 같은 장면 부품이 train/test에 섞여 정보 누수(leakage)가 생기므로, 장면 단위로 분할하고 배경 종류를 stratify + seed 고정해 재현 가능하게 했다.

### 데이터셋 버전 히스토리

| 버전 | 부품 | 배경 | 의도 |
|------|------|------|------|
| v1 | 회색 단색 | 단색 | 기본 형상 |
| v2 | 색·텍스처 랜덤 | 도메인 랜덤화 | 실제 배경 미확정 대응 |
| v2_noisy | v2 동일 | v2 + ToF depth 노이즈 | depth 현실화 |
| v3 | 회색 단색 고정 | 2종(적재형/정렬형) | 형태로 판별 + 빈피킹 2시나리오 정합 |
| **2D 인코더 학습셋** | **회색 단색** | **배경 NaN** | **현재 모델 학습용 1,000 scene** |

### `synth/` 주요 파일

| 파일 | 역할 |

전담 조교가 제공한 모델 파이프라인을 통합한 디렉터리. 팀 6은 이 코드를 우리 데이터로 처음부터 끝까지 직접 구동하며 검증한다. 상세는 `model/INTEGRATION.md` 참고.

## 중간 결과 (실측)

전체 파이프라인을 우리 합성 데이터 1,000장으로 **한 사이클 학습·평가까지 완주**했다.

| 단계 | 결과 |
|------|------|
| 3D CAD Encoder | 27종 식별 정확도 **1.00** → codebook (27, 256) 정상 구축 |
| 2D Depth Detector (test 100 scene / 850 객체) | recall **1.00** · class **80.4%** · CAD 식별 **79.8%** · mask IoU **0.855** · box IoU **0.682** |

- 2단계 학습은 안정적으로 수렴(warmup 후 val loss 7.8→3.7, joint best 3.85).
- 모든 수치는 **depth만 입력**하고 GT 마스크·라벨 없이 낸 결과다.
- 2차 발표는 모델링 진행 단계로, **성능 경쟁이 아니라 파이프라인이 끝까지 동작함을 입증**하는 단계다.

## 실증 & 비교 실험 (진행 중)

- **실물 Depth 촬영** — ToF 카메라로 부품을 촬영해 합성 학습 모델을 실물로 검증하고, 합성↔실물 도메인 갭을 측정한다. (시험촬영 → 적정 밀도·포맷 탐색 → 본촬영)
- **공개 데이터셋 비교** — [BOP ITODD](https://bop.felk.cvut.cz/datasets/)(산업 부품) / IC-BIN(빈피킹 클러터)을 우리 코드 포맷(Depth scene + GT)으로 변환해 동일 지표로 비교한다.
- **솔직한 맹점** — 현재는 데이터셋 비교만 있고 직접 견줄 기존 '모델'이 사실상 없다(동일 과제 = depth-only로 검출·분할 + CAD 식별을 푸는 공개 모델 부재). 추가 조사를 통해 비교 가능한 기준을 찾아 비교 테이블을 채워간다.
