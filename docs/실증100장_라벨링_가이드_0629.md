# 실증 100장 instance 라벨링 가이드 (labelme)

> **목적**: 실증 depth 100장에서 "한 장 안 어느 픽셀이 어느 부품인지"(instance/pixel) 라벨링.
> **방식**: labelme로 부품마다 polygon 그리고 STL 이름 라벨 부여. **각 장은 그룹(g1/g2/g3) 9종 중에서만** 고름.
> **왜 지금**: 미루면 7/9 직전에 100장이 한꺼번에 몰림 + instance가 있으면 그룹 레벨도 자동 도출(상위호환).

## 0. 준비물 (6000 → Mac 전송)

**Mac 터미널에서** (라벨링용 PNG + 정답지 받기):
```bash

scp -P <PORT> -r <USER>@<SERVER_IP>:/data/jtm/synth_out/real_capture100/label_png ~/Desktop/real100_labelme/

# 그룹 정답지(어느 장이 어느 9종인지) + 라벨 목록
scp -P <PORT> <USER>@<SERVER_IP>:/home/jtm/kaist_project/docs/real_capture100_group_labels.json ~/Desktop/real100_labelme/
```

## 1. labelme 설치 (Mac, 1회)
```bash
pip install labelme           # 또는: conda install -c conda-forge labelme
labelme --version             # 확인
```

## 2. 라벨링 실행
```bash
cd ~/Desktop/real100_labelme
labelme label_png --output label_png   # JSON을 PNG 옆에 저장
```
- labelme 창에서 **Create Polygons** (단축키 `Ctrl+N`) → 부품 외곽을 점 찍어 둘러싸고 → 닫으면 라벨 입력창 → **STL 이름** 입력.
- 한 장 끝나면 `Ctrl+S` 저장 → `D`(다음 이미지).
- **배경(좌우 빨강 영역)은 그리지 말 것.** 중앙 부품만.

## 3. ⭐ 라벨 규칙 (가장 중요)
- **각 장은 그 그룹의 9종 중에서만** 나옴. 파일명 `_g1/_g2/_g3`로 그룹 확인.
- 라벨 이름 = **STL 이름** (예: `08_r_guide_a`). `.stl`은 붙여도/빼도 됨(변환기가 처리).
- **같은 부품이 한 장에 2개 있으면** 각각 따로 polygon (instance 분리). 라벨 이름은 같게.
- 회색 단색이라 헷갈리면 → 아래 그룹표 + CAD 컨택트시트(`scratchpad/contact_sheet_27.png`) 대조.

## ⚠️ 주의 / 메모
- **좌표 스케일**: label_png(1696×960) = npy(848×480) **×2**. 변환기가 자동으로 /2 처리.
- **원본 백업**: `/data/jtm/synth_out/real_capture100_BACKUP_ORIGINAL_0629/` (읽기전용).
- **조교 포맷 컨펌 대기 항목** (변환기 `labelme_to_synthformat.py` CONFIG): 해상도 512 / depth meter(÷1000) / 배경 NaN — **합성과 동일하게 가정**해둠. 조교가 다른 값 주면 CONFIG만 수정 후 재변환.
- **6DoF pose 없음**: 실증은 실측 자세 없음(합성엔 quat/euler 있음). instance 마스크 + category_id까지가 실증 라벨의 범위.
- labelme가 무거우면 대안 = CVAT(웹). 단 labelme가 설치·변환 가장 단순.
