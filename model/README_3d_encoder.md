# CAD 3D Encoder: PointNet++ baseline

This package trains a CAD encoder on point clouds generated by `stl_to_pointcloud_dataset.py`.

## Files

```text
cad3d_encoder/
  pointnet2_encoder.py   # pure-PyTorch PointNet++ encoder
  dataset.py             # manifest/npz dataset + two-view augmentation
  losses.py              # supervised contrastive loss
train_cad_encoder.py     # training script
build_cad_memory_bank.py # export CAD embeddings
requirements.txt
```

## Install

```bash
pip install -r requirements.txt
```

## Train

```bash
python train_cad_encoder.py \
  --manifest ./pc_dataset/manifest.json \
  --out_dir ./runs/cad_pointnet2 \
  --epochs 100 \
  --batch_size 16 \
  --n_points 4096 \
  --rotation_aug none
```

Recommended first run for debugging:

```bash
python train_cad_encoder.py \
  --manifest ./pc_dataset/manifest.json \
  --out_dir ./runs/debug_cad_pointnet2 \
  --epochs 2 \
  --batch_size 2 \
  --n_points 1024 \
  --sa1_npoint 256 \
  --sa2_npoint 64 \
  --num_workers 0 \
  --torch_num_threads 1
```

## Rotation augmentation policy

Use `--rotation_aug none` for the first canonical CAD memory bank.
Use `--rotation_aug z` only if rotation around the CAD Z axis should not change CAD identity.
Use `--rotation_aug so3` when training a shape-invariant retrieval embedding.

For pose estimation and local projection alignment, keep canonical tokens in the original CAD coordinate system. A good production design is to maintain two concepts:

```text
shape embedding: rotation-invariant, useful for CAD retrieval
local/canonical tokens: orientation-preserving, useful for 2D-3D projection alignment and pose
```

## Export CAD memory bank

```bash
python build_cad_memory_bank.py \
  --manifest ./pc_dataset/manifest.json \
  --checkpoint ./runs/cad_pointnet2/best.pt \
  --output ./runs/cad_pointnet2/cad_memory_bank.npz \
  --n_points 4096 \
  --save_local
```

Output:

```text
cad_ids       : [M]
class_names   : [M]
embeddings    : [M, D]
local_xyz     : [M, L, 3]      # if --save_local
local_tokens  : [M, L, Dlocal] # if --save_local
```

`embeddings` is used as the CAD memory for later depth-only detection query alignment.
`local_xyz` and `local_tokens` are used for CAD-point-token ↔ image/depth-token projection alignment.
