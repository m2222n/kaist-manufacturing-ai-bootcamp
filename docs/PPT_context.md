## 1. 팀 소개
- 제조 현장의 부품 생산 자동화를 개발하는 팀. 3D 프린터로 출력한 플라스틱 부품을 로봇이 집어 후공정(세척·검사 등)으로 옮기는 자동화를 다룸.
- 부트캠프 프로젝트 = 그중 **"카메라로 부품을 인식하는 비전 AI"** 파트.

## 2. 프로젝트 주제 및 문제 정의
- **주제**: 박스(빈) 안에 담긴 여러 부품을 카메라로 인식하고, 어느 부품인지·어디 있는지 판별해 로봇이 집을 수 있게 하는 비전 시스템 (Bin Picking 인식).
- **해결할 문제**: 부품 종류가 약 28종으로 많고 형상이 비슷한 것도 있음. 박스 안에서 부품이 겹치거나 다양한 자세로 놓임 → 단순 분류로는 부족하고 **3D 구조까지 이해해 부품을 판별**해야 함.
- **왜 중요한가**: 사람이 하던 부품 분류·픽업을 자동화하면 24시간 무인 운영 가능. 실제 산업 현장(로봇 빈피킹) 적용이 목표 = "데이터셋에서만 잘되는 모델"이 아니라 "현장 카메라에서 통하는 모델"이 핵심.
- **프로젝트 방향이 두 번 다듬어진 과정** (정직하게 보여주면 좋음):
  - (1주차) 부품 CAD를 각도별로 렌더링해 데이터셋 구축 + 다객체 인식
  - (2주차) "촬영 사진 vs CAD 렌더 품질 격차"를 직시 → **"실루엣 기반 3D 복원 → 부품 판별"** 로 주제를 구체화. 단순 2D 분류보다 3D 구조로 부품을 구분하는 게 본질이라 판단.

## 3. 데이터 활용 계획
- **자체 데이터 (현재)**: 부품 28종의 CAD(STL) → 자동 렌더링 파이프라인으로 **1,120장** 생성 (28부품 × 40각도). 각도 = 기울임 5단계 × 회전 8방향, 정면 시점, 흰 배경.
  - 장점: GT(원본 3D 형상)를 갖고 있어 복원·인식 품질을 정량 비교 가능.
- **혼합 (계획)**: 3주차부터 **실제 카메라로 박스에 부품을 놓고 촬영** → 합성(CAD 렌더) ↔ 실촬영 사이의 도메인 갭을 메움. (조명·배경·겹침 다양화)
- **공개 데이터셋**: 필요 시 형상 인식 보조용으로 검토.
- 데이터 특징: 부품 형상이 단순 볼록형부터 길고 얇은 형태, 구멍·슬롯 많은 복잡형까지 다양.

## 4. 접근 방식
- **1차 baseline = Visual Hull (Shape-from-Silhouette)**: 여러 각도 실루엣을 역투영해 교집합으로 3D 형상을 복원. 카메라 포즈를 알고(렌더 시 정의) GT 3D를 보유해 정량 평가가 가능한 게 강점.
- **인식 경로 (2안 검토 중)**:
  - SAM으로 객체를 분리 → YOLO Object Detection으로 부품 종류 판별
  - 데이터셋이 충분하면 YOLO Segmentation으로 한 번에
- **부품 판별**: 복원한 3D 형상 ↔ 원본 CAD를 비교(매칭)해 "어느 부품인지" 판정.
- **평가 지표**:
  - 3D 복원: IoU(복원 vs GT 부피 겹침), Chamfer distance(표면 거리), GT가 복원 안에 들어가는 비율
  - 인식: mAP 등 검출/분류 정확도
- **1주차 baseline 정량 결과 (28부품 전체)** — 슬라이드에 표/그래프로:
  - IoU 평균 0.589 (median 0.603, 최소 0.249, 최대 0.813)
  - Chamfer 평균 2.85mm
  - "GT가 복원 안에 들어가는 비율" 평균 0.989 → 복원이 원본을 거의 다 감싼다는 의미
  - **해석 (정직하게)**: 볼록하고 단순한 부품은 잘 복원됨(IoU 0.8+). 길고 얇거나 구멍·오목면이 많은 부품은 Visual Hull 원리상 복원이 부풀어(과대 추정) IoU가 낮음. → Visual Hull은 GT/검증용 baseline으로 두고, 실시간 인식은 별도 경로(소수 뷰 + 학습 기반)로 보완할 계획.

## 5. 이후 일정 계획 (3~6주차)
- 3주차: 실제 카메라 촬영분 추가 (도메인 갭 메우기), YOLO seg/SAM 인식 파이프라인 착수, 데이터 라벨링
- 4주차: 인식 모델 학습 + 검증 (합성 + 실촬영 혼합)
- 5주차: 복원 형상 ↔ CAD 매칭으로 부품 판별 통합, 3D↔2D 비교 방법 적용
- 6주차: **실제 카메라로 부품 인식 라이브 데모** (종착점) + 산업 현장(로봇 빈피킹) 적용 연결
- ⭐ **산업 적용 연결**: 이 프로젝트 산출물(데이터셋·인식 모델·평가 코드·라이브 인식 모듈)은 실제 로봇 빈피킹 시스템의 인식 단계로 이어짐. Visual Hull은 데이터/검증 도구로, 실시간 인식은 학습 기반 모델로 역할을 나눠 현장에 적용.

## 6. 논의·궁금한 점
- 합성(CAD 렌더) ↔ 실촬영 도메인 갭을 줄이는 효과적 방법
- 복원된 3D 형상과 원본 CAD를 비교(매칭)해 부품을 판별하는 방법론 추천
- 박스 안 겹친 부품(occlusion) 분리에 적합한 접근
- 카메라를 소수 뷰만 찍는 실제 환경에서 3D 인식을 어떻게 견고하게 할지

## 작성 시 주의 (중요)
- **회사명·제품명·인물 실명·특정 대기업 고객사·서버 정보 등은 절대 넣지 말 것.** "제조 현장", "협력 산업 현장" 정도의 추상 표현만 사용.
- 톤: 학술 발표체, 간결. 슬라이드당 글머리 3~5개.
- 1주차 baseline 결과(IoU 표)는 **그래프/표 1장**으로 시각화 권장 (잘되는 부품 vs 안되는 부품 대비가 보이게).
- "방향이 두 번 다듬어졌다"는 건 시행착오가 아니라 **문제를 깊이 이해해가는 과정**으로 긍정적으로 표현.
